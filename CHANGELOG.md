- Git parity: SHA-256 object-format repositories are now fully supported, including over the smart HTTP/SSH transport. `extensions.objectFormat = sha256` repositories open (the compatibility gate is lifted), `Version.Init.Init`/`Init_Bare` take an object-format argument, and every object/index/pack/ref/reflog code path is hash-width-aware. The smart transport negotiates the object format (`Version.Upload_Pack.Advertised_Object_Format` reads the advertised `object-format` capability, `Version.Fetch.Remote_Object_Format` discovers it before a clone creates its target, and the receive-pack client sends `object-format=sha256` and repo-width null ids for ref create/delete). - Added `Version.Clone.Clone_Filtered` and `Version.Fetch.Fetch (Remote, Filter_Spec)` for partial clone: HTTP/SSH send the filter when the server advertises the `filter` capability; local transport selectively copies non-blob (or size-bounded) objects via the new `Version.Write.Copy_Object`. The clone is configured as a partial clone, and the local promisor lazy fetch (`Fetch_Object`) now copies only the requested object instead of the whole store.
- Supported the `extensions.worktreeConfig` repository extension: `Version.Config.Read_All` now layers the per-worktree `config.worktree` over the common config (per-worktree values override; absent keys fall back), and `Version.Repository_Format` no longer rejects it.

- Added `Version.Push.Push_Refspec` and `Version.Receive_Pack.Push_Ref`/`Push_Ref_Ssh` (push an arbitrary source commit to a named remote ref over local, HTTP, or SSH, with a fast-forward check unless forced) â€” the engine behind `push SRC:DST`.

- Added a `Force` parameter to `Version.Push.Push` and `Version.Receive_Pack.Push_Branch`/`Push_Branch_Ssh` (and the local branch path) that skips the fast-forward (ancestor) check for non-fast-forward updates.

- Replaced the system-`ssh` subprocess SSH transport with native `ssh_lib`: `Version.Transport.Ssh` now opens an `SSH_Lib.Sessions` session and runs `git-upload-pack`/`git-receive-pack`/`git-lfs-authenticate` over `SSH_Lib.Channels.Open_Exec` (streaming via `Read_Some`/`Write`), with no dependency on an external `ssh` binary.
